# Agent Manifest Specification — v1.0 (Draft / RFC)

**Status:** Request for Comments. Subject to change before v1.0 final.
**License:** MIT.
**Discussion:** open an issue on GitHub.

## 1. Motivation

AI agents today drive web applications the same way humans do — by reading screens and clicking buttons. This is fragile (UI changes break agents), slow (every action is a round-trip), unsafe (no scoped permissions, no human-in-the-loop), and unauditable (no record of which agent did what).

The Agent Manifest is a small JSON document an application publishes at `/.well-known/agent-manifest.json` to declare a structured, machine-readable contract for AI agents.

## 2. Discovery

Conforming applications MUST host the manifest at:

```
https://<host>/.well-known/agent-manifest.json
```

The endpoint MUST return `Content-Type: application/json` and SHOULD be reachable without authentication so that agent runtimes can discover capabilities before presenting credentials.

## 3. Top-level structure

```json
{
  "manifest_version": "1.0",
  "app":             { ... },
  "actions":         [ ... ],
  "approval_rules":  [ ... ],
  "agent_identity":  { ... },
  "audit":           { ... }
}
```

`manifest_version`, `app`, and `actions` are REQUIRED. All other fields are OPTIONAL.

## 4. `app`

| Field         | Type   | Required | Description |
|---------------|--------|----------|-------------|
| `name`        | string | yes      | Human-readable application name. |
| `url`         | URI    | yes      | Canonical app URL. |
| `description` | string | no       | One-paragraph summary. |
| `logo_url`    | URI    | no       | Square logo, served over HTTPS. |

## 5. `actions`

Each action is a discrete operation an agent can request.

| Field                   | Type    | Required | Description |
|-------------------------|---------|----------|-------------|
| `id`                    | string  | yes      | `^[a-z][a-z0-9_]*$` — stable identifier. |
| `description`           | string  | yes      | What the action does, in plain English. |
| `risk`                  | enum    | yes      | `low`, `medium`, or `high`. |
| `scopes`                | array   | no       | Required permission scopes (e.g. `invoices:write`). |
| `requires_approval`     | boolean | no       | If `true`, a human MUST approve every call. Default `false`. |
| `rate_limit_per_minute` | integer | no       | Server-side ceiling. |
| `parameters`            | object  | no       | Map of param name → `{ type, required, description }`. |

### Risk levels

- **low** — read-only, no side effects (e.g. `list_invoices`).
- **medium** — writes or sends communication that is reversible (e.g. `reply_to_ticket`).
- **high** — irreversible, financial, destructive, or PII-affecting (e.g. `refund_invoice`, `delete_customer`).

## 6. `approval_rules`

Rules that escalate specific calls to human approval, even when the action itself does not require approval globally.

```json
{
  "when":    { "action": "refund_invoice", "amount_cents": { "gt": 10000 } },
  "require": "human_approver"
}
```

`require` MUST be one of: `human_approver`, `two_person`, `admin_only`.

The `when` object matches against the request payload. Supported operators in v1.0: `gt`, `gte`, `lt`, `lte`, `eq`. Equality is implicit when the value is a literal.

## 7. `agent_identity`

| Field          | Type    | Default | Description |
|----------------|---------|---------|-------------|
| `required`     | boolean | `true`  | Must the agent identify itself? |
| `verification` | enum    | —       | `bearer_token`, `mtls`, or `signed_jwt`. |

Agents SHOULD present a stable identity URI (e.g. `agent://openai/operator`) that the application logs alongside every call.

## 8. `audit`

| Field            | Type    | Default | Description |
|------------------|---------|---------|-------------|
| `log_all`        | boolean | `true`  | Application commits to logging every agent call. |
| `retention_days` | integer | —       | How long logs are retained. |
| `export_format`  | enum    | —       | `json`, `csv`, or `ndjson`. |

## 9. Security considerations

- **Default-deny.** Anything not declared in `actions` MUST be rejected by the runtime.
- **No secrets in manifests.** The manifest is public.
- **Stable IDs.** Action `id`s are part of the API contract; renaming is a breaking change.
- **Versioning.** Breaking schema changes bump `manifest_version` (e.g. `2.0`). Additive changes use the same major.

## 10. Validation

The canonical JSON Schema is published at `schema/manifest.v1.json` and mirrored at <https://agentgate.lovable.app/schema/manifest.v1.json>. A hosted validator is available at <https://agentgate.lovable.app/validator>.

## 11. Reference implementation

A working enforcement runtime (`/v1/invoke`) that consumes this manifest — including scope checks, approval queueing, and immutable audit logs — is provided by [AgentGate](https://agentgate.lovable.app) and is open to inspection.
